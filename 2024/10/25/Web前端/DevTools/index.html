基本功能

DOM

文档对象模型（DOM）是一个网络文档的编程接口，它代表页面，以便程序可以改变文档的结构、风格、内容。DOM将文档表示为节点和对象，这样，编程语言就可以与页面交互。

DOM是用于建立网站的Web API。

$0、$1、$2、$3、$4

$0 是对当前在 Element 面板中选中的 html 节点的引用，$1是对上一次选择节点的引用，以此类推直到$4，通过$的引用可以读取 DOM 节点的一些属性。

$ 和 $$

$ 等价于 document.querySelector()

$$ 等价于 document.querySelectorAll()

$_

$_为对上次输出结果的引用

应用Application

应用面板主要用于管理和调试Web应用的数据和资源，通过Application面板，开发者可以查看和操作各种存储机制、缓存数据、服务工作者以及应用安全设置等。

存储

LocalStorage

查看数据

是一种在浏览器中保存键值对数据的机制，数据没有过期时间

，可长期保存在用于浏览器中。使用LocoalStorage可以看到当前所有的存储数据。

添加/修改/删除数据

1、可以直接在面板中添加新的键值对，或修改现有的数据。

2、选中要删除的键值对，右键选择删除，或使用删除键

SessionStorage

它的数据只在当前会话有效，一旦关闭页面或浏览器，数据就会被清除。SessionStorage适用于存储一些临时数据，如表单状态、用户操作记录等。

添加/修改/删除数据同LoacalStorage

Cookies

HTTP Cookie - HTTP | MDN (mozilla.org)

Cookies是Web开发中最早的存储机制之一，用于在客户端保存小块数据。Cookies通常用于会话管理、个性化设置以及跟踪用户行为等。

可添加、修改、阐述Cookies

源代码Source

文件资源管理器（左侧）

文件树：显示当前项目的文件和目录结构

搜索栏：用于快速搜索文件或内容

代码编辑器：查看和编辑代码

Snippets代码片段

允许保存并运行JavaScript代码片段，这对于反复执行的代码或测试特定功能非常有用。

调试控制

继续F8：继续执行代码，直到遇到下一个断点或代码执行完毕。

逐行执行F10：执行下一行代码，但不进入函数内部。如果当前行是一个函数调用，则直接执行该函数，跳过内部详细执行。

逐步进入F11：进入函数内部，逐行执行函数内部代码。

逐步跳出Shift+F11：执行完当前函数内部的所有代码，然后返回到函数调用点的下一行代码。

查看调用栈

调用栈是显示当前代码执行路径的工具，可以帮助了解函数是如何被调用的，以及当前执行到哪一行代码。

#在代码暂停执行时，调用栈面板会显示当前的调用路径

#调用栈按顺序列出所有被调用的函数，从最新的函数调用开始，最上方是当前执行的函数，依次向下是之前调用的函数

#点击调用栈中的任意函数，可以导航到相应的代码位置，查看和调试该函数的具体实现

作用域

显示当前上下文中的所有变量及其值，帮助了解在当前执行点上哪些变量是可访问的，以及他们的具体状态

Local Scope本地作用域

显示当前函数内部的所有局部变量

通过展开变量，可以查看其具体值和结构（如对象的属性）

Global Scope全局作用域

显示全局范围内的变量，通常包括窗口对象（Window）和全局变量

了解全局变量的状态，可以帮助分析在整个应用程序范围内的变量变化

Closure闭包

闭包：内部函数总是可以访问其所在的外部函数中声明的参数和变量，即使在其外部函数被返回了

如果当前执行的函数是一个闭包，作用域面板会显示该函数捕获的外部变量

分析闭包中的变量，有助于理解函数在其定义时所捕获的上下文

监视变量与表达式

在动态调试中，可以实时跟踪特定变量或表达式的值的变化。

添加监视表达式

#在作用于面板中，右键点击某个变量并选择“添加到监视”

#或者在“监视”面板中，点击“添加表达式”，然后输入想监视的变量名或表达式

查看监视表达式结果

#在“监视”面板中，可以看到所有添加的监视表达式以及当前的值

#在代码执行到断点时，监视面板会自动更新表达式的值，帮助实时跟踪变量状态。

请求列表

显示了所有被记录的网络请求，每个请求以一行的形式展示，包含以下信息：请求类型、状态码、请求和响应时间、请求大小等

请求详情图

点击请求列表中的某个请求，可以打开其详细信息视图

底部功能区

提供了网络请求的总体统计信息和概览

网络请求的基本信息

启动和停止网络活动记录

默认情况下，网络面板会自动记录所有网络活动，点击该按钮可以手动控制记录的开始和停止

![](./../../imgs/DevTools/465d2c92fcd4b7b71d06aa4a7b163d15 (1).png)

清除网络活动记录

清除当前的网络请求记录，使网络面板重新开始记录新的网络请求

请求的详细信息

Headers（请求头和响应头）

显示请求和响应的所有头信息，包括URL、方法、状态码、服务器等，以及查询字符串参数

Preview（响应预览）

以可读格式响应数据，适用于JSON、HTNML等格式

Response（响应体）

显示原始响应数据，便于查看和分析

Timing（请求的时间信息）

显示请求各个阶段的时间小号，如发起请求、等待响应、接收数据等，帮助分析性能瓶颈

Cookies

列出请求和响应中涉及的所有Cookies信息，包括名称、值、域、路径、过期时间等

过滤和搜索

调试和分析网页的网络请求时，网络面板提供了强大的过滤和搜索功能，帮助快速定位特定的请求和数据。

过滤器

可以从大量的网络请求中筛选特定类型的请求，便于分析和调试。

文本过滤器Text Filter

#在工具栏的过滤器输入框中输入文本，可以实时过滤出名称或URL中包含该文本的请求

正则表达式过滤器Regular Expression Filter

#在文本过滤器中使用正则表达式进行高级筛选。输入内容以“/”包围即可识别为正则表达式

如：/api/

类型过滤器Type Filter

#在工具栏中可以选择不同类型的请求进行筛选，如文档、样式表、脚本、图片、Image、XHR、Fetch等

状态码过滤器Status Code Filter

#输入特定的HTTP状态码（如200、404）或状态码范围（如3xx、4xx）来筛选请求

分析网络性能

首字节时间（TTFB）

从发起请求到接收到第一个字节响应数据的时间。TTFB反映了服务器处理请求并开始传输数据所需的时间，是衡量服务器性能的重要标志

内容加载时间

指资源从开始请求到全部内容加载完成所需的时间，这一指标反映了资源的整体下载速度。

DOM内容加载时间

指浏览器完成解析HTML文档、构建DOM树，并且没有样式表、图片等资源阻塞时的时间点。

完全加载时间

模拟网络条件

模拟慢速网络

在设置限制和，所有网络请求都会根据选择的速度进行模拟，帮助测试网页在慢速网络环境下的加载性能

模拟离线状态

离线模式下，所有网络请求都会失败，模拟用户在无网络连接的情况，测试网页的离线功能和缓存机制（如Service Worker和离线缓存）。

强制重新加载和禁用缓存

调试过程中，浏览器缓存可能会影响请求结果。DevTools提供了强制重新加载和禁用缓存的功能，以确保每次请求都从服务器获取最新数据。

元素审查Elements

面板介绍

DOM树视图

显示当前页面的HTML结构，以树状视图呈现。开发者可以展开、折叠节点，查看和编辑HTML元素及其特性。

样式视图Styles

显示选定元素的CSS样式规则，开发者可以查看、修改、添加和删除CSS属性，还可以禁用特定的样式规则进行调试

计算样式

显示选定元素的所有计算样式，列出所有应用于该元素的最终CSS属性值，帮助开发者了解样式的来源和优先级

盒模型

可视化显示选定元素的盒模型，包括内容区域、填充、边框和外边距。开发者可以直观地调整这些属性，优化页面布局。

事件监听器

列出选定元素的所有事件监听器，可以查看和调试这些事件的绑定情况

DOM断点

允许开发者在DOM变化时触发断点，例如节点的属性更改、节点移除或子节点变化，帮助调试动态DOM变化

属性

显示和编辑选定元素的 HTML 属性。开发者可以直接双击属性名称或值进行编辑，实时更新页面内容。

将鼠标移动到网页中的元素上。该元素会被选中并高亮显示。

实时编辑 HTML 结构

修改和添加元素属性

在需要编辑DOM上右键，在菜单中选择 添加属性 或 修改属性 即可

CSS 样式编辑

查看、修改元素和添加、删除、禁用 CSS 规则

选中DOM元素后，在样式视图中可以看到其对应的样式，在需要修改的样式上双击修改即可

同上选中DOM元素后，在需要添加、删除和禁用样式上双击修改即可。

盒模型与尺寸调整

盒模型（Box Model）是网页布局和设计中的核心概念，它描述了一个 HTML 元素的内容以及周围的空间。

盒模型由以下四个部分组成，从内到外依次是：内容区（Content）、内边距（Padding）、边框（Border）、外边距（Margin）

在需要修改的数据上，双击编辑即可。

DOM 断点

设置 DOM 断点进行调试

只需要设置 DOM 断点的元素上，右键选择 发生中断的条件 ，按需要选择规则即可

查看和管理 DOM 断点

在DOM 断点窗体中，可以看到全部的 DOM 断点 ，可以在需要的断点上右键进行更多操作

事件监听

在事件监听面板上找到需要查看事件，展开即可查看监听此事件的函数。可以移除事件或是查看具体的函数位置。
